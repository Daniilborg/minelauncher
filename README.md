# My Launcher — свой Minecraft-клиент на Fabric

## Что уже готово
- Каркас JavaFX-приложения: экран входа → главный экран (Играть / Магазин / Моды / Настройки)
- Полный вход через Microsoft (device code flow) + офлайн-режим для разработки без Azure
- На вкладке "Играть": свой никнейм (сохраняется), выбор релиза или снапшота из полного списка версий Mojang
- Реальный запуск игры на Fabric: скачивание версии и библиотек (манифест Mojang), установка Fabric loader (Fabric meta API), скачивание ассетов, распаковка нативных библиотек и запуск отдельным процессом
- Магазин модов/ресурспаков/шейдеров на базе Modrinth API: поиск, карточка с описанием, установка в один клик
- Вкладка "Моды": список установленного, вкл/выкл (без удаления файла), удаление
- Вкладка "Настройки": память JVM, свой путь к Java, разрешение окна — сохраняется в `~/.my-launcher/settings.json`
- Тёмная тема

## Исправлено
- Downloader раньше при неудачной загрузке файла тихо писал ошибку в консоль и считал файл скачанным — из-за этого магазин мог показать "Установлено ✓", хотя файл не долетел. Теперь любая неудачная загрузка (хоть в магазине, хоть при скачивании версии игры) даёт понятную ошибку в UI
- launch() раньше считал запуск успешным сразу после старта процесса — если игра падала мгновенно, лаунчер всё равно писал "Игра запущена!". Теперь ждём 3 секунды и проверяем, что процесс всё ещё жив
- У каждой версии теперь своя папка (`~/.my-launcher/instances/<версия>/`) с отдельными mods/resourcepacks/shaderpacks — раньше все версии делили одну общую папку, из-за чего мод под одну версию мог остаться в mods/ и сломать запуск другой (Fabric видит несовместимость и совершенно правильно откажется стартовать — см. следующий пункт)
- **Дубли библиотек на classpath** ("duplicate ASM classes found" и подобное): ванильная версия и профиль Fabric иногда указывают одну и ту же библиотеку с разными версиями (например, asm 9.6 у одного источника и 9.10.1 у другого) — раньше обе просто клеились в один classpath. Теперь библиотеки дедуплицируются по group/artifact, остаётся версия новее
- **Выбор версии на вкладке "Играть" сбрасывался** при переключении на другую вкладку и обратно — вкладка пересобиралась с нуля каждый раз, включая ComboBox'ы версии, и всегда возвращалась к последнему релизу. Из-за этого могло казаться, что моды "пропали" из вкладки "Моды" — на самом деле просто версия тихо поменялась под капотом. Теперь вкладка "Играть" строится один раз и переиспользуется, как и остальные
- **Автопоиск подходящей Java**: если версии игры нужна Java новее, чем та, на которой работает лаунчер, и в настройках не указан свой путь — лаунчер сам поищет подходящую JDK в стандартных местах установки (`/Library/Java/JavaVirtualMachines` на Mac, `/usr/lib/jvm` на Linux, `Program Files\Java` и аналоги на Windows) и использует её автоматически

## Про переход на папки по версиям
Из-за перехода на отдельные папки под каждую версию все моды, поставленные раньше через магазин, нужно поставить заново (уже находясь на нужной версии) — старые файлы остались в больше не используемой общей папке `instance/`.

## Важно: версии Minecraft 26.x требуют Java 25
С 2026 года Mojang сменила схему версий: `1.21.11` была последней версией со старой нумерацией (1.x), дальше идут `26.1`, `26.2`, `26.3`... (год.дроп; снапшоты вида `26.3-snapshot-2`, альфы вида `26.3-alpha.2`). **Все версии 26.x и новее требуют минимум Java 25.**

Старые версии (1.21.x и ниже) продолжают работать на Java 17/21 как обычно.

Я не могу поставить Java на твой компьютер — у меня нет к нему доступа, только к своей песочнице. Но после того, как ты её поставишь, лаунчер сам её найдёт (см. автопоиск выше) — руками в настройках указывать путь не придётся. Проще всего поставить с [adoptium.net](https://adoptium.net): выбери Java 25, macOS, aarch64 (раз у тебя Apple Silicon), скачай `.pkg` и запусти — он ставит именно в `/Library/Java/JavaVirtualMachines/`, ровно то место, которое лаунчер и сканирует.

## Про никнейм и UUID
- Никнейм на вкладке "Играть" можно менять свободно — сохраняется и подставляется при следующем запуске
- Для офлайн-профиля смена никнейма пересчитывает UUID так же, как это делает сама игра в офлайн-режиме (`OfflinePlayer:<никнейм>`) — иначе сохранения/данные игрока не совпадут с новым именем
- Для настоящего Microsoft-аккаунта UUID не трогается (он привязан к реальному профилю) — меняется только то, что видно как имя в самой игре

## Как устроен магазин
- Источник — api.modrinth.com (бесплатный, без ключа), одним поиском покрывает моды, ресурспаки и шейдеры (project_type)
- Поиск фильтруется под версию игры, указанную на вкладке "Играть"; для модов дополнительно фильтруется под Fabric
- "Установить" качает файл сразу в нужную папку: `mods/`, `resourcepacks/` или `shaderpacks/` внутри `~/.my-launcher/instance/`
- Моды после установки сразу видно во вкладке "Моды" с переключателем вкл/выкл (Fabric подхватывает всё из `mods/`)
- Ресурспаки и шейдеры в лаунчере не переключаются — они просто оказываются в нужной папке, а какой из них активен, выбираешь как обычно внутри игры (Настройки → Ресурспаки / Видео → Шейдеры). Для шейдеров нужен ещё мод Iris (или OptiFine) — без него пункта "Шейдеры" в видеонастройках не будет

## Как устроен запуск игры
- Все файлы (версии, библиотеки, ассеты) хранятся в `~/.my-launcher/`, отдельно от обычного Minecraft Launcher — они не конфликтуют
- Сама игра (сохранения, моды, resource-паки) живёт в `~/.my-launcher/instance/`
- По кнопке "Играть" лаунчер: берёт версию из поля (по умолчанию — последний релиз), находит для неё последнюю стабильную версию Fabric, докачивает недостающие файлы (уже скачанные — пропускает по sha1) и запускает `java` отдельным процессом
- На экране входа есть кнопка "Пропустить (офлайн, для разработки)" — заходит с фиктивным профилем (`DevPlayer`, `userType=legacy`), без Microsoft/Xbox. Скачивание версии, Fabric и запуск работают так же; недоступен только официальный онлайн-режим (мультиплеер на серверах с проверкой аккаунта, скины и т.д.) — это ожидаемо, это чисто для разработки и теста лаунчера, а не способ играть без покупки игры
- Память JVM пока фиксирована (1024–4096 МБ) — станет настраиваемой во вкладке "Настройки"
- Логи игры сейчас пишутся в консоль самого лаунчера (Run-вкладка в IntelliJ)

### Известные ограничения (пока не реализовано)
- Не проверяется/не подтягивается автоматически нужная версия Java под конкретную версию игры (например, 1.20.5+ требует Java 21) — лаунчер только предупредит в статусе, если текущая Java младше нужной
- Нет поддержки очень старых версий (до 1.14) — их и Fabric не поддерживает
- Нет argument-файлов для очень длинного classpath на Windows (на практике с обычным набором модов до этого не доходит)

## Как запустить
1. Открой папку проекта в IntelliJ IDEA как Gradle-проект (File → Open → выбрать папку `my-launcher`)
2. Дай Gradle подтянуть зависимости (нужен интернет и Java 17+, IntelliJ сам предложит нужный JDK)
3. Запусти `App.java` через зелёную стрелку рядом с `main`, либо `./gradlew run`

## Что нужно сделать, прежде чем логин заработает
1. Зайди на [portal.azure.com](https://portal.azure.com) → App registrations → New registration
   - Supported account types: **Personal Microsoft accounts only**
   - Redirect URI можно не указывать (используется device code flow)
   - Скопируй **Application (client) ID**
2. **Важно:** с 2023 года Microsoft требует отдельного одобрения новых Azure-приложений для доступа к Xbox Live / Minecraft API. Без одобрения будет ошибка `Invalid app registration, see https://aka.ms/AppRegInfo`. По этой ссылке есть описание и форма заявки на доступ. Одобрение не мгновенное, так что стоит подать заявку сразу, а пока разрабатывать всё остальное — тестировать вход прямо сейчас не обязательно.
   - Приложения, зарегистрированные до этого ограничения, работают без заявки — если у тебя уже есть такое, можно использовать его client_id.
3. Вставь client_id в `MicrosoftAuth.java` вместо `YOUR_AZURE_CLIENT_ID`
